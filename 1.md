# 기획서

## 백준 연동 소크라테스식 AI 문제풀이 튜터 (Chrome Extension)

---

## 1. 문제 정의 (Why)

### 1.1 현재 문제

* 백준 문제 풀이 과정에서:

  * 막히면 **해설/블로그/ChatGPT로 바로 답 확인**
  * 사고 과정 붕괴
  * 학습 효율 급감
* 기존 AI 도구:

  * **정답 중심**
  * “도움”이 아니라 “대체”

### 1.2 핵심 문제 요약

> **사용자는 ‘힌트’를 원하지만, 시스템은 ‘답’을 준다.**

이 미스매치를 해결하지 않으면
아무리 AI를 붙여도 **학습 도구가 아니라 치트 도구**가 된다.

---

## 2. 목표 정의 (What)

### 2.1 핵심 목표

* 백준 문제를 **대신 풀어주지 않는다**
* 사용자가 **생각하도록 강제한다**
* **웹페이지 안에서 바로** 작동한다

### 2.2 비목표 (명확히 배제)

* 자동 문제 해결 ❌
* 자동 코드 생성 ❌
* 자동 제출 ❌
* “정답 빠르게 보기” ❌

---

## 3. 서비스 한 줄 정의

> **백준 문제 페이지에서, 사용자가 원할 때만
> 힌트 → 사고 유도 → 설명 → 접근 → (최종) 답을
> 단계적으로 제공하는 AI 튜터 크롬 익스텐션**

---

## 4. 타겟 사용자

### 4.1 Primary

* 대학생 / 고등학생
* 알고리즘 학습자 (백준 기준)

### 4.2 Secondary

* 코딩테스트 준비 취준생
* 알고리즘 스터디 참여자

### 4.3 의도적 제외

* 정답만 필요한 사용자
* 부정행위 목적 사용자

👉 **이 서비스는 모두를 위한 게 아니다.**
학습 의지가 없는 사용자는 애초에 타겟 아님.

---

## 5. 핵심 UX 설계 (가장 중요)

### 5.1 위치

* 백준 문제 페이지 내부
* 플로팅 패널 또는 사이드 패널

### 5.2 단계 구조 (강제)

| 단계      | 사용자 액션    | 시스템 제공        |
| ------- | --------- | ------------- |
| Level 0 | 자동        | 문제 요약 / 제약 정리 |
| Level 1 | 버튼 클릭     | 방향 힌트         |
| Level 2 | 버튼 클릭     | 사고 유도 질문      |
| Level 3 | 버튼 클릭     | 핵심 아이디어       |
| Level 4 | 버튼 클릭     | 접근법 / 의사코드    |
| Level 5 | **명시 요청** | 정답 코드         |

**규칙**

* 자동 진행 ❌
* 단계 건너뛰기 ❌
* Level 5 전에는 코드 ❌

---

## 6. 소크라테스 강제 로직 (핵심 차별점)

### 6.1 프롬프트만으로는 실패한다

* LLM은 **항상 친절하게 답을 주려는 경향**이 있음
* “정답 주지 마”는 지켜지지 않는다

### 6.2 해결 방식

**서버 단 강제 통제**

* 요청마다 `level` 명시
* level < 5:

  * 코드 블록 생성 차단
  * 구현 디테일 필터링
* 응답 후 **검증 단계**

  * 코드 패턴 감지 시 재요청 또는 삭제

👉 **이 통제 없으면 이 프로젝트는 망한다.**

## 7. 기술 아키텍처

### 7.1 전체 구조

```
Baekjoon Page
  └─ Chrome Extension
        └─ 문제 DOM 추출
              ↓
          Backend API
              ↓
      Embedding (OpenAI)
              ↓
        Vector Search
              ↓
        LLM (OpenAI)
```

---

### 7.2 기술 스택

#### 프론트엔드

* Chrome Extension (Manifest V3)
* Vanilla JS / TypeScript
* DOM Parsing (백준 구조 고정 활용)

#### 백엔드

* FastAPI (Python)
* REST API
* 문제별 세션 관리

#### AI

* LLM: OpenAI (`gpt-4o-mini`)
* Embedding: OpenAI `text-embedding-3-*`
* RAG:

  * 알고리즘 개념
  * 문제 유형 패턴
  * 사고 프레임
  * ❌ 정답 코드 데이터

## 11. 실패 시나리오 (중요)

이 중 하나라도 방치하면 실패한다.

1. **정답 노출이 너무 쉽다**
2. 소크라테스 단계가 형식적이다
3. 백준 정책/윤리 이슈 발생
4. “와 신기하다” 이후 재사용 안 됨

---

## 12. 성공 기준 (MVP)

* 사용자가 **정답 보기 전에 페이지를 떠나지 않는다**
* 힌트 단계에서 **체류 시간 증가**
* “답 보기” 사용 비율이 **50% 이하**
